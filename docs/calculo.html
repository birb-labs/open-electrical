Cálculo & circuitos

Os dois serviços de engenharia do plugin: o cálculo luminotécnico (método dos lúmens, NBR 5413) que posiciona as luminárias, e a distribuição automática de circuitos (NBR 5410) que agrupa os elementos em circuitos e dimensiona condutor/disjuntor.

EL-CALC-LIGHT — método dos lúmens

src/commands/ElementCommands.cpp — calcLightingAuto() → runLightingCalc() → placeLightsInRoom()

O comando lista os ambientes para o usuário escolher um (ou "Todos"), pergunta a luminária (fluxo/potência) e roda o método dos lúmens por ambiente, usando os parâmetros do próprio ambiente (refletâncias, fator de manutenção, plano de trabalho, contraste — nada é global). As luminárias são inseridas numa grade uniforme recortada pelo polígono do cômodo.

Nome do comando. O único comando registrado é EL-CALC-LIGHT. A função interna runLightingCalc() aceita um rótulo de undo (EL-LIGHT-AUTO era o rótulo previsto para um segundo comando), mas não há um comando EL-LIGHT-AUTO na tabela de Commands.cpp — digitar esse nome resulta em "Unknown command". Para expor um alias, adicione a linha correspondente na tabela.
A iluminância-alvo é derivada do uso do ambiente + contraste (NBR 5413), não digitada. Pré-requisito: existir ao menos um ambiente (EL-ROOM) com uso/pé-direito definidos.

A fórmula (LightingCalculator)

EL-CIRCUIT / EL-CIRCUIT-AUTO — distribuição de circuitos

src/commands/CircuitCommands.cpp — distributeCircuitsAuto()

Chama CircuitDistributor::distribute() (serviço puro, que só mexe no modelo) e, de volta ao comando, empurra o número de circuito atribuído para o atributo CIRCUITO de cada símbolo já desenhado (via syncElementAttributes). Sem esse passo, os símbolos ficariam mostrando o que foi digitado na inserção, ou "?".

DistributionResult r = CircuitDistributor::distribute(project);
for (auto& e : project.elements) {
    if (e->circuitId < 0 || e->handle.empty()) continue;
    const std::string label = std::to_string(e->circuitId);
    if (e->circuitLabel == label) continue;   // já mostra este circuito
    e->circuitLabel = label;
    syncElementAttributes(*e);                 // reescreve o atributo CIRCUITO no desenho
}

EL-CIRCUIT (manual) hoje apenas anuncia que usará os tetos padrão da NBR 5410 e delega para o automático — o ponto de extensão para uma UI de arrastar elementos entre circuitos.

Regras aplicadas (CircuitDistributor)

src/services/CircuitDistributor.h / .cpp
  • Iluminação e tomadas nunca no mesmo circuito.
  • Iluminação agrupada até um teto de VA convencional (maxLightingVA = 1200).
  • Tomadas de uso geral (TUG) agrupadas até VA/contagem (maxOutletVA = 1270, maxOutletsPerCircuit = 10).
  • Cada tomada de uso específico (TUE — chuveiro, A/C…) recebe o seu próprio circuito.
  • Seção de condutor e disjuntor seguem a escada mínima padrão (minConductorMM2(), breakerFor()).
  • As fases são balanceadas em round-robin entre os condutores vivos disponíveis.

Como customizar a distribuição

Quero mudar…Onde mexer
Tetos de VA / máximo de tomadas por circuitoDistributionOptions (maxLightingVA, maxOutletVA, maxOutletsPerCircuit)
Seção mínima de condutor por tipo de circuitoCircuitDistributor::minConductorMM2()
Escolha do disjuntor por seçãoCircuitDistributor::breakerFor()
Quantos condutores (F/N/PE/R) cada tipo carregaconductorsFor() em WireCounts.h — impacta a fiação (veja EL-WIRE)
Quadro que recebe os novos circuitosDistributionOptions.targetPanelId (-1 = quadro principal/geral)
Cálculo da queda de tensãoCircuitDistributor::computeVoltageDrops() (ρ do cobre, k por nº de fases)

Vínculo quadro↔circuito e queda de tensão

Cada quadro tem um id próprio (Panel::id) e cada circuito aponta para o seu quadro via Circuit::panelId. Após distribuir, o CircuitDistributor reconstrói a lista Panel::circuitIds de cada quadro — é ela que preenche o atributo CIRCUITOS do símbolo do quadro e o agrupamento do unifilar.

A queda de tensão por circuito (Circuit::voltageDropPct) é calculada por computeVoltageDrops(), chamada dentro de EL-WIRE-AUTO (quando os comprimentos dos eletrodutos já são conhecidos). O modelo é simplificado — NBR 5410, cobre, carga resistiva:

ΔV = k · ρ · L · I / S        ΔV% = 100 · ΔV / V_ref
  ρ = 0,0172 Ω·mm²/m (cobre)   k = 2 (mono/bifásico) | √3 (trifásico)
  L = soma dos eletrodutos do circuito   I = carga / V_ref   S = seção da fase
Simplificado: ignora fator de potência e derating por temperatura/agrupamento. Para precisão, substitua pelas tabelas completas da NBR 5410.
Tipos de circuito reservados. A distribuição automática cria apenas Lighting, GeneralOutlets e SpecificOutlets. Os tipos Motor e Mixed estão reservados: as contagens de condutor (WireCounts.h) e o dimensionamento já existem, mas ainda não há caminho de criação automática — só surgem se um circuito for marcado manualmente / por um comando futuro.
Ordem recomendada de trabalho: EL-ROOM → EL-CALC-LIGHT / colocar tomadas → EL-CIRCUIT-AUTO → EL-CONDUIT-AUTO → EL-WIRE-AUTO. Cada etapa depende dos dados da anterior.
