Eletrodutos & roteamento

Como o plugin desenha os eletrodutos (o caminho físico) — automática ou manualmente — e como cada extremidade "gruda" no contorno do componente pela ElementOutline, para nunca começar/terminar dentro de um símbolo.

EL-CONDUIT-AUTO — roteamento automático

src/commands/CircuitCommands.cpp — routeConduitAuto()

Pré-requisito: existir circuitos (EL-CIRCUIT-AUTO). Pergunta se deve omitir os eletrodutos horizontais (só descidas verticais — um estilo comum de documentação), monta as RoutingOptions e chama Router::route(). Os eletrodutos são desenhados como polilinhas 3D, para que as descidas verticais (teto → dispositivo, que só diferem em Z) fiquem visíveis em vez de colapsar num ponto.

RoutingOptions opt;
opt.omitHorizontal = /* resposta Yes/No */;
opt.material       = project.settings.conduitMaterial;
opt.ceilingZ       = /* pé-direito do ambiente mais alto, em unidades de desenho */;
RoutingResult r = Router::route(project, opt);
for (const auto& c : project.conduits)
    if (c.handle.empty()) c.handle = drawPolyline3D(kLayerConduit, c.path);
O Router conecta os elementos de cada circuito de volta ao seu quadro: em modo completo, trechos ortogonais no teto + descidas verticais; em omitHorizontal, apenas a descida vertical de cada dispositivo.

EL-CONDUIT — eletroduto manual entre dois elementos

src/commands/CircuitCommands.cpp — routeConduit()

Selecione a ORIGEM e o DESTINO. O eletroduto corre reto entre os dois pontos de conexão definidos pelo contorno de cada componente: começa no contorno da origem no ponto mais próximo do destino, e termina no contorno do destino no ponto mais próximo da origem — assim nunca começa/termina dentro de um símbolo. Os circuitos que passam por um trecho manual são os circuitos dos dispositivos das pontas.

AttachSpec src, dst;                       // tipo + nº de módulos + transformada de cada pick
mutualAttachPoints(src, dst, pA, pB);      // pontos de conexão mutuamente mais próximos
Conduit c;
c.srcHandle = src.handle; c.dstHandle = dst.handle;
c.path = { pA, pB };                       // segmento reto, pontas no contorno
c.handle = drawPolyline(kLayerConduit, c.path, false);

EL-CONDUIT-EDIT — remodelar um trecho manual

src/commands/CircuitCommands.cpp — editConduit()

Escolha o trecho e depois [Source/Destination/Curve]:

  • Curve (padrão): informe um ponto por onde a curva deve passar; o bulge é calculado pela flecha (sagitta) do ponto em relação à corda — bulge = 2·sag / corda = tan(θ/4). O trecho vira um arco.
  • Source / Destination: mova aquela extremidade; o novo ponto é reancorado no contorno do respectivo componente (attachPoint), continuando dentro das conexões permitidas.

A edição atualiza o desenho e o modelo (path/bulge/length), então flui direto para a lista de materiais. Só trechos retos elemento-a-elemento (path.size() == 2) são editáveis.

EL-ROUTE-AUTO — rotear + fiar num passo

src/commands/CircuitCommands.cpp — routeAllAuto()

Atalho: roda routeConduitAuto() e, na sequência, Router::pullWires() + refreshWiringBalloons(), deixando a lista de materiais completa (eletroduto + condutores + balões) num único comando.

Pontos de conexão — ElementOutline

src/utilities/ElementOutline.h / .cpp

É a peça que decide onde um eletroduto pode tocar cada componente. É deliberadamente livre de BRX/AcGe (opera sobre Point3 puro), então é unit-testável. Cada tipo expõe um contorno que espelha a geometria da SymbolFactory:

KindContorno
LightCírculo do raio da luminária de teto
OutletCadeia de triângulos de uma tomada baixa, com o mesmo nº de módulos
SwitchCírculo de interruptor de 1 seção (+ a haste da parede)
PanelRetângulo 3:1

O contorno vive no referencial local do símbolo (+Y para fora da parede, origem no ponto de inserção); os chamadores convertem para/de coordenadas do mundo com a posição + rotação da referência de bloco (toWorld/toLocal).

// ponto do contorno (no mundo) mais próximo de um alvo
Point3 nearestAttachPoint(Kind kind, int modules, const Point3& origin,
                          double rot, const Point3& target);

// duas pontas convergindo (usado pelo EL-CONDUIT manual):
pA = attachPoint(a, b.pos);   // ponto de A mais perto de B
pB = attachPoint(b, pA);      // ponto de B mais perto de pA
pA = attachPoint(a, pB);      // refina até ambos concordarem
pB = attachPoint(b, pA);
