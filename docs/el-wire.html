EL-WIRE / EL-WIRE-AUTO

O comando de distribuição de fiação: lê os circuitos, calcula quantos condutores passam por cada eletroduto e desenha o "balão" (aro + haste + trilho + símbolos de Fase / Neutro / Terra / Retorno) dentro de cada trecho.

Visão geral

Há dois comandos, quase idênticos:

  • EL-WIRE-AUTO — faz todo o trabalho: puxa os fios no modelo e (re)desenha os balões em todos os eletrodutos.
  • EL-WIRE — apenas imprime uma mensagem e chama EL-WIRE-AUTO; existe para dar um nome "sem sufixo" ao usuário.
Pré-requisitos no desenho: já existir pelo menos um circuito (EL-CIRCUIT-AUTO) e os eletrodutos (EL-CONDUIT-AUTO). Um eletroduto sem circuito associado não carrega condutores e, portanto, não recebe balão.

Fluxo do comando

src/commands/CircuitCommands.cpp — routeWireAuto() (trecho simplificado) 
void routeWireAuto() {
    ProjectData& project = ProjectContext::instance().project();
    if (project.conduits.empty()) { acutPrintf(_T("\nNo conduits yet...")); return; }

    UndoGroup undo(_T("EL-WIRE-AUTO"));
    Router::pullWires(project);                 // 1. calcula os fios (modelo)
    const int drawn = refreshWiringBalloons(project);  // 2. (re)desenha os balões
    ProjectContext::instance().save();          // 3. persiste

    // Soma o comprimento de todos os condutores puxados (para o resumo).
    double totalWire = 0.0;
    for (const auto& c : project.conduits)
        for (const auto& w : c.wires) totalWire += w.lengthM;

    acutPrintf(_T("\nEL-WIRE: %d conduit(s), %d circuit(s), %d wiring tag(s) drawn (%.2f m).\n"),
               (int)project.conduits.size(), (int)project.circuits.size(), drawn, totalWire);
}
Trecho simplificado: o fonte também imprime uma dica quando drawn == 0 (rode EL-CIRCUIT-AUTO antes).

A linha impressa é o seu diagnóstico: se aparecer 0 circuit(s) ou 0 wiring tag(s), o problema é de dados (rode EL-CIRCUIT-AUTO antes), não de desenho.

Idempotência e "troncos" (trunk union)

refreshWiringBalloons primeiro apaga os balões antigos (guardados em Conduit::annotationHandles) e então agrupa eletrodutos que se sobrepõem no mesmo ponto médio — assim um trecho compartilhado por 2+ circuitos mostra um balão com a soma dos circuitos.

src/commands/CircuitCommands.cpp — refreshWiringBalloons() 
// apaga balões anteriores (sem órfãos)
for (auto& c : project.conduits) {
    for (const auto& h : c.annotationHandles) { AcDbObjectId id;
        if (entityIdByHandle(h, id)) eraseEntity(id); }
    c.annotationHandles.clear();
}
// agrupa trechos coincidentes e desenha 1 balão com a UNIÃO dos circuitos

Contagem de condutores (por circuito)

A regra-chave: o número de componentes de um circuito não multiplica os condutores; o que soma é o número de circuitos distintos que passam pelo eletroduto. A quantidade de fios de cada circuito depende do seu tipo:

src/services/WireCounts.h 
struct ConductorSet { int phase, neutral, ground, retorno; /* + operator+= e total() */ };

inline ConductorSet conductorsFor(CircuitType type) {
    switch (type) {
        case CircuitType::Lighting:        return { 1, 1, 0, 1 };  // F + N + Retorno
        case CircuitType::GeneralOutlets:  return { 1, 1, 1, 0 };  // F + N + Terra
        case CircuitType::SpecificOutlets: return { 1, 1, 1, 0 };
        case CircuitType::Motor:           return { 1, 1, 1, 0 };
        case CircuitType::Mixed:           return { 1, 1, 1, 1 };
    }
    return { 1, 1, 1, 0 };
}

Quais circuitos passam por um eletroduto e a soma total dos condutores são resolvidos no Router:

src/services/Router.cpp 
// circuitos que passam por um eletroduto: os seus próprios circuitIds,
// ou (trechos antigos) os circuitos dos componentes das pontas.
std::vector<int> Router::conduitCircuits(const ProjectData&, const Conduit&);

// soma de conductorsFor(tipo) sobre cada circuito distinto do eletroduto
ConductorSet Router::conduitConductors(const ProjectData& p, const Conduit& c) {
    ConductorSet total;
    for (int cid : conduitCircuits(p, c))
        total += conductorsFor(circuitById(p, cid)->type);
    return total;
}

Anatomia do balão

Todo o desenho do balão vive numa única função: drawWiringBalloon(project, conduit). Ela é composta por: aro (o "C" no eletroduto) → haste → trilho branco → um símbolo por condutor. Tudo é posicionado a partir do ponto médio M do eletroduto e escalado à unidade dos símbolos (~0.09).

src/commands/CircuitCommands.cpp — drawWiringBalloon() (métricas) 
const double s     = c.mirror ? -1.0 : 1.0;  // espelhamento horizontal (EL-WIRE-FLIP)
const double ringR = 0.028;   // raio do aro (o "C")
const double tH    = 0.05;    // meia-altura do símbolo vertical
const double barW  = 0.02;    // largura dos retângulos verticais (todos iguais)
const double hc    = 0.045;   // meio-comprimento das barras horizontais (T / L)
const double sp    = 0.13;    // espaçamento entre símbolos
const double railY = M.y + ringR + 0.085;   // altura do trilho

O aro (o "C")

É um arco de ~300° (abertura de ~60°) cuja "boca" aponta para a haste/trilho. Diminuir ringR encolhe o aro; aumentar gapHalf abre mais o "C".

const double gapHalf = 30.0 * kPi / 180.0;         // metade da abertura → "C" mais/menos fechado
const double a0 = gapHalf, a1 = 2.0 * kPi - gapHalf;
for (int i = 0; i <= 16; ++i) {
    const double a = a0 + (a1 - a0) * i / 16;
    arc.emplace_back(M.x + s * ringR * std::cos(a), M.y + ringR * std::sin(a), 0.0);
}
H.push_back(drawColoredPolyline(kLayerWiring, arc, 255, 255, 255));  // branco

Haste + trilho

const double railX0  = M.x + s * (ringR + 0.03);
const double railLen = cs.total() * sp + 0.05;      // comprimento ∝ nº de símbolos
const double railX1  = railX0 + s * railLen;
// haste: do aro ao trilho ; trilho: linha branca horizontal
drawColoredPolyline(kLayerWiring, { {M.x + s*ringR, M.y}, {railX0, railY} }, 255,255,255);
drawColoredPolyline(kLayerWiring, { {railX0, railY},      {railX1, railY} }, 255,255,255);

Formato de cada simbologia

Cada condutor é um glifo volumétrico (retângulos preenchidos via drawSolidRect). Dois helpers desenham as barras:

auto vbar = [&](double y0, double y1, int r,int g,int b){   // barra vertical
    drawSolidRect(kLayerWiring, x, (y0+y1)/2, barW/2, |y1-y0|/2, r,g,b); };
auto hbar = [&](double x0, double x1, double y, int r,int g,int b){ // barra horizontal
    drawSolidRect(kLayerWiring, (x0+x1)/2, y, |x1-x0|/2, barW/2, r,g,b); };
src/commands/CircuitCommands.cpp — drawWiringBalloon() (glifos) 
auto glyph = [&](int r,int g,int b, int kind) {
    switch (kind) {
        case 0:  vbar(railY - tH, railY + tH, r,g,b); break;                  // Fase
        case 1:  vbar(railY - tH, railY + tH, r,g,b);                          // Neutro
                 hbar(x, x - s*2.0*hc, railY + tH, r,g,b); break;             //   topo → esquerda
        case 2:  vbar(railY, railY + tH, r,g,b); break;                        // Retorno (metade sup.)
        case 3:  vbar(railY - tH, railY + tH, r,g,b);                          // Terra
                 hbar(x - hc, x + hc, railY + tH, r,g,b); break;              //   topo centralizado
    }
    x += s * sp;
};
for (int i=0;i<cs.phase;  ++i) glyph(255, 0,   0,   0);  // F  vermelho
for (int i=0;i<cs.neutral;++i) glyph(0,   255, 255, 1);  // N  ciano
for (int i=0;i<cs.ground; ++i) glyph(0,   255, 0,   3);  // PE verde
for (int i=0;i<cs.retorno;++i) glyph(255, 255, 255, 2);  // R  branco

Atenção — renderização. Use drawSolidRect (AcDbSolid, preenchido) para as barras. Uma tentativa anterior com polilinha de largura constante (setConstantWidth) fazia o EL-WIRE abortar e nenhum balão aparecia. drawSolidRect é o mesmo primitivo de preenchimento dos símbolos das tomadas e renderiza de forma confiável.

EL-WIRE-FLIP (espelhar o balão)

Seleciona um eletroduto e espelha o balão para o outro lado do trecho (útil para o balão não colidir com geometria vizinha). Alterna Conduit::mirror (persistido) e redesenha só aquele balão — todas as coordenadas x são multiplicadas por s = mirror ? -1 : 1.

src/commands/CircuitCommands.cpp — flipWiring() 
Conduit* c = conduitByEntity(project, id);      // eletroduto escolhido
c->mirror = !c->mirror;                          // alterna o espelho
/* apaga os handles antigos */ c->annotationHandles.clear();
drawWiringBalloon(project, *c);                  // redesenha espelhado
ProjectContext::instance().save();
