Relatórios & remoção

Os comandos que produzem documentos (legenda, quadro de cargas, lista de materiais, diagrama unifilar) e os que removem objetos mantendo modelo e desenho em sincronia — mais o EL-UNDO.

EL-REPORT — os quatro documentos

src/commands/ReportCommands.cpp — generateReport()

Pergunta qual documento [Legend/Loads/Bom/Sld/All], pede um ponto de inserção e coloca cada relatório como um bloco de MText na camada de texto, empilhados verticalmente. O texto vem do serviço ReportGenerator (que só produz strings — quem desenha é a camada de comando).

OpçãoDocumentoGerador
LegendLegenda: símbolo → descrição de cada tipo usado no projetoReportGenerator::legend()
LoadsQuadro de cargas: uma linha por circuito com dimensionamento (inclui a coluna V-drop %, a queda de tensão calculada por EL-WIRE-AUTO)ReportGenerator::loadSchedule()
BomLista de materiais: quantidades agregadas de dispositivos, eletroduto e fioReportGenerator::billOfMaterialsText()
SldDiagrama unifilar (netlist textual): cada circuito sob o seu quadro (por panelId), com seção, disjuntor, carga e queda de tensãoReportGenerator::singleLineDiagram()
AllEmite os quatro, empilhados (passo dy = 4.0)—

EL-LOAD-SCHEDULE — quadro de cargas direto

src/commands/ReportCommands.cpp — generateLoadSchedule()

Atalho para inserir só o quadro de cargas num ponto escolhido (ReportGenerator::loadSchedule()).

EL-SINGLE-LINE — diagrama unifilar direto

src/commands/ReportCommands.cpp — generateSingleLine()

Atalho para inserir só o unifilar (ReportGenerator::singleLineDiagram()).

O unifilar lista cada circuito uma única vez, sob o quadro a que pertence (comparação por Circuit::panelId == Panel::id). Circuitos cujo panelId não corresponde a nenhum quadro (dados antigos) são atribuídos ao quadro principal. Para isso funcionar, insira os quadros com EL-PANEL (que aloca o Panel::id) e rode EL-CIRCUIT-AUTO.
Cada relatório é montado a partir do estado atual de ProjectData. Para relatórios corretos, rode antes a distribuição (EL-CIRCUIT-AUTO), o roteamento e a fiação — o quadro de cargas e a BOM leem circuitos, eletrodutos e condutores.

EL-DEL — remover componentes/eletrodutos

src/commands/DeleteCommands.cpp — deleteSelected()

Selecione qualquer quantidade de componentes e/ou eletrodutos; cada entidade é apagada do desenho e removida do modelo, para que o quadro de cargas e a lista de materiais permaneçam em sincronia. Ao remover um eletroduto, o seu balão de fiação também é apagado (via annotationHandles), evitando etiquetas órfãs.

if (dropFromModel(project, handleOfId(id))) ++modelHits;   // tira do modelo (+ apaga balão)
if (eraseEntity(id)) ++erased;                              // apaga a entidade do desenho

EL-DEL-ROOM — remover só o contorno de um ambiente

src/commands/DeleteCommands.cpp — deleteRoom()

Escolha um ambiente numa lista; apaga apenas a polilinha de contorno. Os componentes ficam onde estão — só o vínculo com o ambiente é limpo (e->roomId = -1), para que nada aponte para um ambiente que não existe mais.

EL-UNDO — desfazer a última ação do plugin

src/commands/ReportCommands.cpp — undoLast()

Reverte o último grupo de undo do plugin no desenho e recarrega o modelo a partir do .dwg restaurado (ProjectContext::instance().reload()), para que modelo e desenho voltem juntos. Todos os comandos abrem um UndoGroup nomeado (ex.: UndoGroup undo(_T("EL-ROOM"))), o que também os torna reversíveis pelo UNDO nativo do BricsCAD.
