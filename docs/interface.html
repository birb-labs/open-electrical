Interface, configuração & i18n

A camada visual do plugin: a paleta dockável (wxWidgets), o diálogo de configuração do projeto (EL-CONFIG) e o sistema de tradução em tempo de execução. Tudo em src/ui/.

EL — abrir/fechar a paleta principal

src/commands/ConfigCommands.cpp — showPalette()

O comando EL alterna a paleta dockável (ui::togglePalette()). A paleta é um wxPanel de botões de comando agrupados por etapa do fluxo de trabalho.

src/ui/MainPalette.h / .cpp 
class MainPalettePanel : public wxPanel {
    void refresh();                          // reconstrói conteúdo dinâmico (resumo do projeto)
    void run(const wxString& command);       // envia um comando à linha de comando do documento ativo
    wxButton* addButton(wxSizer*, const std::string& labelKey, const wxString& command);
};

Cada botão é criado com addButton(sizer, chaveDeTexto, "EL-XXX "): o rótulo é uma chave de tradução (i18n) e a ação apenas digita o comando na linha de comando. Ou seja, adicionar um botão novo = uma linha, e ele reaproveita toda a lógica do comando.

Hospedagem por plataforma (PaletteHost)

src/ui/PaletteHost.cpp / .h

O painel é idêntico nos dois sistemas; só a hospedagem muda: no Windows ele vive numa docking pane BcUiPanelMFC; no Linux é acoplado como um wxWindow nativo. Esse isolamento (mais o version-script do wxWidgets) foi o que resolveu os travamentos anteriores do wx no Linux.

EL-CONFIG — configuração do projeto

src/commands/ConfigCommands.cpp — configProject()

Carrega as configurações atuais, abre o diálogo modal (ui::showConfigDialog), e persiste. Uma troca de idioma tem efeito imediato em toda a UI.

ProjectSettings edited = project.settings;
if (ui::showConfigDialog(edited)) {
    project.settings = edited;
    Localization::instance().setLanguage(edited.uiLanguage);   // idioma muda na hora
    ProjectContext::instance().save();
    ui::refreshPalette();
}

Internacionalização (i18n)

src/ui/Localization.h / .cpp · resources/lang/*.json

Regra de ouro: comandos e identificadores internos são SEMPRE em inglês; só os textos da UI são traduzidos. Os pacotes de idioma são mapas JSON planos (resources/lang/<código>.json: "en", "pt-BR", "es"). O inglês é o fallback embutido: qualquer chave ausente no pacote ativo cai para o inglês, e qualquer chave ausente em todos os pacotes retorna a si mesma.

// no código da UI:
std::string  EL_TR ("btn.cancel");   // UTF-8, idioma ativo
std::wstring EL_TRW("msg.done");     // std::wstring para a camada wx/BRX

Localization::instance().setLanguage("pt-BR");   // troca em tempo de execução

Adicionar um idioma ou uma string

  1. Nova string: use uma chave (ex.: "btn.route") no código com EL_TR/EL_TRW e adicione a chave em cada resources/lang/*.json.
  2. Novo idioma: crie resources/lang/<código>.json (mapa plano chave→texto), acrescente o código à lista de idiomas do diálogo de config, e ele será carregado por loadPacks().
