Elementos & símbolos

Os comandos que criam objetos no desenho: ambientes, pontos de luz, tomadas, interruptores e quadros. Cada elemento é um bloco NBR 5444 gerado programaticamente pela SymbolFactory, ligado a um registro no modelo (ProjectData) pelo handle da entidade.

Todo comando desta página cria um objeto do modelo, insere o bloco correspondente e chama ProjectContext::instance().save() — isto é o que mantém o desenho e o modelo sincronizados após salvar/reabrir o .dwg.

EL-ROOM — demarcar um ambiente

src/commands/RoomCommands.cpp — insertRoom()

Clique os cantos do ambiente um a um (mínimo 3). Uma linha elástica segue o cursor a partir do canto anterior; Enter fecha o contorno e a palavra-chave Undo remove o último canto. O plugin desenha a polilinha fechada na camada EL_ROOMS, calcula a área e abre o diálogo do ambiente (nome, uso, função, pé-direito e os parâmetros luminotécnicos).

Room room;
room.id       = project.allocRoomId();
room.boundary = corners;               // cantos coletados por pickCorners()
room.recomputeArea(project.settings.unit);
room.boundaryHandle = drawPolyline(kLayerRooms, corners, /*closed*/ true);
if (!ui::showRoomDialog(room)) { project.nextRoomId--; return; }   // cancelar = rollback do id
O ambiente é a base de tudo: EL-CALC-LIGHT, EL-OUTLET-AUTO e EL-SWITCH-AUTO iteram sobre project.rooms. Sem ambiente, esses comandos avisam "No rooms defined" e não fazem nada.

EL-LIGHT — ponto de luz manual

src/commands/ElementCommands.cpp — insertLight()

Escolhe o ponto, abre showLightDialog (teto ou arandela, potência, letra de comando, circuito) e insere o bloco. Pontos de teto são referenciados ao pé-direito (NBR 5444): o bloco é rebaixado a Z = pé-direito − offsetFromCeiling, lido do ambiente sob o ponto (ou 2,8 m padrão). Arandelas giram para a perpendicular da parede via pickRotationOnWall.

O texto do bloco (atributos) vem de attrsFor(): a luminária de teto grava POTENCIA, COMANDO e CIRCUITO (o número do circuito é desenhado entre hifens, ex.: -3-); a arandela grava só comando + circuito.

EL-OUTLET / EL-OUTLET-AUTO — tomadas

src/commands/ElementCommands.cpp — insertOutlet() / insertOutletAuto()

EL-OUTLET (manual): ponto → showOutletDialog (tipo, finalidade, altura, VA) → gira na parede → insere. O bloco é escolhido por outletBlockFor(): combos tomada+interruptor usam blocos fixos; as demais são uma cadeia de 1–3 triângulos nomeada pelas alturas de montagem de cada módulo (convenção baixa/média/alta da NBR 5444).

EL-OUTLET-AUTO: aplica o critério simplificado da NBR 5410 por ambiente — ≥ 1 tomada a cada 5 m de perímetro; áreas molhadas (cozinha/área de serviço) a cada 3,5 m — e distribui os pontos ao longo do contorno:

double spacing = (usage == Kitchen || usage == ServiceArea) ? 3.5 : 5.0;
int    count   = std::max(1, (int)(perimetroM / spacing));
// para cada k: posiciona no ponto (k+0.5)*passo ao longo do perímetro
O critério automático é simplificado (só o espaçamento de perímetro). Regras por cômodo (bancada de cozinha a cada 3,5 m independentemente da área, box de banheiro, etc.) ainda não estão implementadas — ajuste em insertOutletAuto().

EL-SWITCH / EL-SWITCH-AUTO — interruptores

src/commands/ElementCommands.cpp — insertSwitch() / insertSwitchAuto()

EL-SWITCH (manual): ponto → showSwitchDialog → parede → insere. O bloco depende do SwitchKind: 1/2/3 seções, paralelo (three-way), intermediário (four-way) e campainha (switchBlock()). Cada seção vira um atributo COMANDO<n>.

EL-SWITCH-AUTO: coloca um interruptor de 1 seção junto à primeira abertura (porta) de cada ambiente — ou, na falta, num vértice do contorno — e o vincula aos pontos de luz daquele ambiente (controlledLightHandles).

EL-PANEL — quadro de distribuição

src/commands/ElementCommands.cpp — insertPanel()

O primeiro quadro do projeto é assumido como o geral (QGBT); os seguintes, como quadros de distribuição (QD). showPanelDialog permite editar nome e o tipo (aparente EL_PANEL_SURFACE ou embutido EL_PANEL_FLUSH).

bool anyPanel = /* já existe algum Panel? */;
cfg.isMain = !anyPanel;               // 1º quadro = QGBT
cfg.name   = cfg.isMain ? "QGBT" : "QD";

EL-EDIT — editar um componente já inserido

src/commands/ElementCommands.cpp — editElement()

Selecione um componente do plugin; ele reabre o diálogo daquele tipo (aplicado a uma cópia, para que Cancelar não altere nada) e grava as mudanças no modelo, no dicionário e no símbolo. A regra de redesenho é a chave:

SituaçãoO que EL-EDIT faz
O novo bloco é o mesmo (só mudou texto/potência/circuito)Só reescreve os atributos updateBlockAttributes(id, attrs) — a referência é preservada
A geometria muda (ex.: interruptor ganha seção, tomada ganha módulo)Apaga a entidade e recria o bloco na mesma posição e rotação
A mesma função syncElementAttributes() é usada por EL-CIRCUIT-AUTO para empurrar o número de circuito recém-atribuído para o atributo CIRCUITO dos símbolos já desenhados.

Como os símbolos são gerados (SymbolFactory)

src/services/SymbolFactory.h / .cpp

Em vez de um .dwg de símbolos, os blocos são definidos no próprio desenho sob demanda. SymbolFactory::ensure(db, nome) cria o bloco se ainda não existir, então insertSymbol() sempre encontra um símbolo real. Convenção local de todos os símbolos: +Y aponta para dentro do ambiente (a parede é a linha y=0); a rotação para casar com a parede real acontece na inserção (rotação da referência), não na geometria.

kOutletLow/Med/HighEL_OUTLET_L / _M / _HTriângulo; preenchimento = altura (vazio/meia/cheia)
cadeia dinâmicaEL_OUTLET_L_M[_H]…1–3 triângulos encadeados ápice-a-base
kSwitch1/2/3, kSwitch3Way/4Way, kSwitchBellEL_SWITCH_*Círculo deslocado da parede com haste; seções = fatias

Versionamento de geometria — o campo mais importante ao editar símbolos

Sempre que alterar a geometria de qualquer símbolo, incremente SymbolFactory::kSymbolVersion. Cada bloco criado carrega esse número como XData. Ao carregar, ensure() compara o carimbo do bloco (que vive dentro do .dwg, não do .lrx) com kSymbolVersion e redefine automaticamente qualquer bloco de uma build antiga — é o que faz a nova geometria aparecer em desenhos que já contêm os blocos velhos. Esquecer disso = o usuário continua vendo o símbolo antigo mesmo após recompilar.
